wget (World Wide Web Get)adalah semacam aplikasi browse tapi tanpa tampilan visual hanya berbasis teks dan kemampuan dasarnya adalah mengambil data dari sebuah url, wget ini mampu mengambil seluruh data berapapun besarnya dan uniknya jika saat mendownload kemudian laptop mati seluruh proses tidak akan diulang kita bisa melanjutkannya dengan penambahan option -c (--continue)dalam bebrapa kali tutorial linux sering kali melihat wget dan curl saya kira ini adalah perintah yang sama untuk mengambil data ternyata berbeda curl (singkatan dari Client URL) fungsinya tidak hanya digunakan untuk mengunduh file, tetapi juga untuk mengirim data, mengecek header keamanan, bahkan berinteraksi dengan API (Application Programming Interface), saya masih bingung dengan perbedaannnya curl dengan wget mereka sama sama bisa mengambil data, jika wget dianalogikan sebagai truk pengangkut yang hanya fokus mengambil dan menyimpan kurir curl ini seperti kurir bisa mengambil peket juga bisa mengirim, wget pun sebernannya bisa mengirim paket namun wget ini ibarat truk yang didesain untuk mengangkut barang besar ada ada kerbatasan seperti tidak selincah kurir, wget bisa saja mengirim data dengan menggunkan metode POST utuk hal hal sederhana seperti mengisi form login atau mengirim data teks pendek sedangkan curl adalah ahli kiri mengirim bisa mengirim file berupa gambar, dokument, json sebaai data aplikasi curl juga bisa berkomunikasi dengan API tapi itu nanti saja menurut saya iu praktek di tingkatan yang melebihi pemahaman dasar saya, kesimpulan sederhana pada misalkan saya berhasil masuk ke server sebuah ponsel jika saya ingin mengambil seluruh data pada ponsel maka sangat disarankan untuk menggunkan wget, namun jika saya ingin menyisipkan semacam file ke ponsel tersebut disinilah curl digunakan,
jadi jika di terjemahkan perintah wget -q -O - https://archive.kali.org/archive-key.asc | sudo apt-key add -
wget (Jalankan program pengambil data dari internet) -q (Mode "Quiet" agar tidak muncul laporan download di layar) -O (Simpan hasil unduhan ke lokasi tertentu) - (Simpan ke "layar/output" saja, bukan jadi file di harddisk) https://archive.kali.org/archive-key.asc (Alamat kunci digital milik Kali Linux yang mau diambil) | (Simbol "Pipa" untuk menyalurkan isi kunci tadi ke perintah berikutnya) sudo (Minta izin Admin karena akan mengubah sistem keamanan) apt-key (Program pengelola kunci keamanan aplikasi) add (Perintah untuk menambahkan kunci baru ke dalam daftar percaya) - (Baca kunci dari "saluran pipa" yang tadi dikirim oleh wget)
